PM Exercises to Try

Frameworks and exercises I use — and recommend to anyone sharpening their product thinking. Pick one, time-box it, write it up.

🎯 Beginner
Product Strategy

Define a Product Vision & Strategy

Pick any product you use daily. Write a 1-page vision, identify the target segment, define the value prop, and outline a 6-month roadmap with 3 key bets.

⏱ 2–3 hours📄 Vision doc + roadmap
Vision → Strategy → Roadmap
⚖️ Beginner
Prioritisation

MoSCoW Prioritisation Exercise

Take a backlog of 15–20 features for a hypothetical product. Categorise as Must/Should/Could/Won't Have. Justify each with business rationale.

⏱ 1–2 hours📄 Prioritised backlog
MoSCoW Framework
🔍 Intermediate
User Research

User Journey Mapping

Map the end-to-end journey of a student applying to study abroad — from awareness to visa approval. Identify touchpoints, emotions, pain points, and opportunities.

⏱ 3–4 hours📄 Journey map
Stages → Actions → Emotions → Opportunities
📊 Intermediate
Metrics & Analytics

Define a Metrics Framework (AARRR)

Choose a product. Define its North Star Metric, then build a full AARRR funnel with specific KPIs and measurement methods for each stage.

⏱ 2–3 hours📄 Metrics tree
AARRR Pirate Metrics + North Star
🧩 Intermediate
Problem Framing

Root Cause Analysis with 5 Whys

Take a product failure scenario (e.g. "checkout abandonment up 20%"). Apply 5 Whys to trace from symptom to root cause. Write a hypothesis and solution.

⏱ 1–2 hours📄 5 Whys + hypothesis
5 Whys → Fishbone → Hypothesis
✏️ Advanced
Product Design

Design a Feature from Scratch

Design a "smart document checklist" for a student visa platform. Write the user story, acceptance criteria, edge cases, wireframe, and success metrics.

⏱ 4–5 hours📄 User story + AC + wireframe
User Story → AC → Wireframe → Metrics
